# Итоговое исследование временного ряда: задача №4

## Назначение репозитория

Этот репозиторий содержит результат задачи №4: итоговый пайплайн прогнозирования временного ряда потребления электроэнергии, результаты его статистического тестирования, тестирования производительности и итоговый отчёт исследования.

## Описание временного ряда

Анализируется почасовой временной ряд потребления электроэнергии. Временная метка — `DateTime`, целевая переменная — `Consumption`. Данные охватывают период с 2019-01-01 00:00:00 по 2024-03-31 23:00:00. По результатам EDA выявлены суточная и недельная сезонности.

## Постановка задачи

Цель — построить offline-пайплайн прогноза потребления электроэнергии на горизонте 168 часов. Основная метрика — MAE. Дополнительно используются RMSE, sMAPE и MASE.

## Данные

Используются подготовленные выборки из задачи №1:

- `data/processed/electricity_train.csv`;
- `data/processed/electricity_test.csv`.

Повторное train/test-разделение в задаче №4 не выполняется.

## Выбор итогового пайплайна

По результатам задачи №3 лучшей ML/DL-моделью стала `MLForecast_GradientBoosting_lags`. В задаче №4 в качестве итоговой реализации используется `GradientBoostingRegressor` как быстрый вариант градиентного бустинга с лаговыми, скользящими и календарными признаками.

Компоненты пайплайна:

1. загрузка готовых train/test-выборок;
2. формирование лагов `1`, `2`, `3`, `24`, `48`, `168`;
3. расчёт скользящих средних и стандартных отклонений за 24 и 168 часов;
4. добавление календарных признаков;
5. обучение градиентного бустинга;
6. рекурсивный прогноз на 168 часов;
7. расчёт метрик качества;
8. статистическое тестирование;
9. тестирование производительности.

## Анализ аномалий

В задаче №3 были рассмотрены `IQR`, `Rolling robust z-score` и `IsolationForest`. Наиболее интерпретируемым методом для данного ряда выбран `Rolling robust z-score`, потому что он учитывает локальный временной контекст и не помечает слишком большую долю наблюдений как аномальные.
